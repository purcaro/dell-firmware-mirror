Prerequisites

Before running Terraform you need the following in place:

RequirementNotes
Terraform ≥ 1.5Covered in step 4
Proxmox VE 9 hostReachable over HTTPS on port 8006
Proxmox API tokenCovered in step 2
Snippets content enabledCovered in step 3 — required for cloud-init
SSH public keyWill be injected into the VM via cloud-init
Internet access from ProxmoxNeeded to download the Ubuntu cloud image
ℹ️

The Terraform scripts live in the terraform/ directory of the dell-firmware-mirror repo. All commands below are run from that directory unless noted otherwise.

2

Create a Proxmox API Token

SSH into your Proxmox host (or open a root shell in the Proxmox web console) and run the following commands to create a dedicated terraform user and an API token:

# Create a dedicated user for Terraform
pveum user add terraform@pam

# Create a role with the minimum required privileges
pveum role add TerraformRole -privs \
  "Datastore.AllocateSpace Datastore.AllocateTemplate Datastore.Audit \
   Pool.Allocate Sys.Audit Sys.Console Sys.Modify \
   VM.Allocate VM.Audit VM.Clone VM.Config.CDROM VM.Config.Cloudinit \
   VM.Config.CPU VM.Config.Disk VM.Config.HWType VM.Config.Memory \
   VM.Config.Network VM.Config.Options VM.Migrate VM.Monitor VM.PowerMgmt"

# Assign the role to the user (/ = datacenter root)
pveum aclmod / -user terraform@pam -role TerraformRole

# Create an API token (--privsep=0 lets the token inherit the user's ACLs)
pveum user token add terraform@pam terraform --privsep=0

The last command prints something like:

┌──────────────┬──────────────────────────────────────┐
│ key          │ value                                │
╞══════════════╪══════════════════════════════════════╡
│ full-tokenid │ terraform@pam!terraform              │
│ value        │ xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx │
└──────────────┴──────────────────────────────────────┘

Your proxmox_api_token variable value is the two parts joined with =:
terraform@pam!terraform=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

⚠️

Copy the token secret now — Proxmox only displays it once. If you lose it, delete the token and create a new one.

3

Enable Snippets on the Datastore

The Terraform script uploads a cloud-init file as a Proxmox snippet. The target datastore must have the Snippets content type enabled:

  1. In the Proxmox web UI, go to Datacenter → Storage
  2. Click on local (or whichever datastore matches snippets_datastore in your tfvars)
  3. Click Edit
  4. In the Content field, add Snippets
  5. Click OK
ℹ️

You can also do this from the shell: pvesm set local --content vztmpl,backup,iso,snippets

4

Install Terraform

Run this on the machine where you'll execute Terraform (your workstation, a jump host, etc.).

Ubuntu / Debian:

wget -O - https://apt.releases.hashicorp.com/gpg | \
  sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg

echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] \
  https://apt.releases.hashicorp.com $(lsb_release -cs) main" | \
  sudo tee /etc/apt/sources.list.d/hashicorp.list

sudo apt update && sudo apt install -y terraform

Verify:

terraform version
Terraform v1.9.x
ℹ️

On macOS use brew install terraform. On Windows use winget install Hashicorp.Terraform.

Configure Variables

Copy the example vars file and edit it:

cd terraform/
cp terraform.tfvars.example terraform.tfvars
$EDITOR terraform.tfvars

The table below describes every variable. The three marked required have no default and must be set.

VariableDefaultDescription
proxmox_endpointrequiredProxmox HTTPS URL, e.g. https://192.168.1.10:8006
proxmox_api_tokenrequiredToken from step 2, e.g. terraform@pam!terraform=xxx
ssh_public_keyrequiredYour SSH public key string (contents of ~/.ssh/id_ed25519.pub)
proxmox_insecuretrueSkip TLS cert check — fine for self-signed Proxmox certs
proxmox_nodepveProxmox node name (shown in top-left of web UI)
iso_datastorelocalWhere the Ubuntu cloud image is stored
snippets_datastorelocalWhere the cloud-init snippet is stored
vm_datastorelocal-lvmWhere VM disks live
vm_namedell-firmware-mirrorVM display name in Proxmox
vm_id200Proxmox VM ID — must not already be in use
cpu_cores2Number of vCPUs
memory_mb4096RAM in MiB
os_disk_gb20Root OS disk size in GiB
mirror_disk_gb200Dedicated data disk for firmware files
network_bridgevmbr0Proxmox network bridge
vm_ipv4_cidremptyStatic IP in CIDR notation; leave blank for DHCP
vm_gatewayemptyDefault gateway when using a static IP
repo_url…purcaro/dell-firmware-mirror.gitHTTPS repo URL cloned onto the VM

A minimal terraform.tfvars looks like this:

proxmox_endpoint  = "https://10.0.0.5:8006"
proxmox_api_token = "terraform@pam!terraform=abc12345-..."
proxmox_node      = "pve"
ssh_public_key    = "ssh-ed25519 AAAAC3Nza... user@host"

Init & Apply

Initialise — downloads the bpg/proxmox provider (~30 MB, one time only):

terraform init

Initializing the backend...
Initializing provider plugins...
- Finding bpg/proxmox versions matching "~> 0.73"...
- Installing bpg/proxmox v0.73.0...

Terraform has been successfully initialized!

Plan — preview changes without making them:

terraform plan

Plan: 3 to add, 0 to change, 0 to destroy.

  + proxmox_virtual_environment_download_file.ubuntu_noble
  + proxmox_virtual_environment_file.cloud_init
  + proxmox_virtual_environment_vm.dell_firmware_mirror

Apply — create everything:

terraform apply

Terraform shows the same plan and asks for confirmation:

Do you want to perform these actions?
  Terraform will perform the actions described above.
  Only 'yes' will be accepted to approve.

  Enter a value: yes

proxmox_virtual_environment_download_file.ubuntu_noble: Creating...
proxmox_virtual_environment_file.cloud_init: Creating...
proxmox_virtual_environment_vm.dell_firmware_mirror: Creating...
proxmox_virtual_environment_vm.dell_firmware_mirror: Still creating... [10s elapsed]
...
Apply complete! Resources: 3 added, 0 changed, 0 destroyed.

Outputs:

vm_id              = 200
vm_ipv4_addresses  = [["10.0.0.50"]]
ℹ️

The Ubuntu cloud image download happens on the Proxmox host itself and only runs once — subsequent applies skip it because overwrite = false.

⏱️

After the VM starts, cloud-init takes ~2–3 minutes to finish installing Docker, cloning the repo, and starting services. The VM is reachable via SSH before cloud-init finishes, but services won't be up yet.

Verify the VM

Use the IP printed in the Terraform output (or find it in the Proxmox web UI under the VM's Summary tab).

SSH in:

ssh ubuntu@<VM_IP>

Check cloud-init completed successfully:

cloud-init status --wait
status: done

Check the nginx container is running:

docker ps
CONTAINER ID   IMAGE                   COMMAND                  STATUS
a3f1d2e8c901   dell-firmware-mirror    "/docker-entrypoint.…"   Up 2 minutes

Check the systemd services:

systemctl status dell-firmware-mirror.service
● dell-firmware-mirror.service - Dell Firmware Mirror (nginx via Docker)
     Loaded: loaded (/etc/systemd/system/dell-firmware-mirror.service; enabled)
     Active: active (running) since ...

systemctl status dellmirror-sync.timer
● dellmirror-sync.timer - Daily Dell firmware mirror sync (02:00)
     Active: active (waiting) ...

Tail the nginx access log:

journalctl -u dell-firmware-mirror -f

Run the mirror sync manually (first-time population):

cd /opt/dell-firmware-mirror
./domirror.sh
[Thread-1] Downloading BIOS_xxx_R740.exe ...
[Thread-2] Downloading iDRAC_xxx.exe ...

Test that nginx serves files:

curl -I http://<VM_IP>:8080/
HTTP/1.1 200 OK
Server: nginx/1.27.x
✅

Point your iDRAC / Lifecycle Controller clients at http://<VM_IP>:8080 as the firmware repository URL.

8

Remote Management Scripts

The scripts/ directory contains four scripts you can run from any machine with SSH access to the VM — no need to log in manually.

ScriptWhat it does
watch-access.shStream live nginx access log (HTTP requests as they arrive)
watch-errors.shStream live nginx error and warning lines only
mirror-sync.shRun the firmware sync and stream its coloured output; accepts extra flags
vm-status.shOne-shot health snapshot: Docker container, services, disk usage, last sync

One-time setup — tell the scripts where your VM is:

# Recommended: gitignored local file so your IP never ends up in git
echo 'VM_HOST="<VM_IP>"' > scripts/config.local.sh

# Or edit directly
$EDITOR scripts/config.sh   # set VM_HOST (and optionally SSH_KEY)

If your SSH key is not the default (~/.ssh/id_rsa / id_ed25519), also set:

echo 'SSH_KEY="~/.ssh/my_proxmox_key"' >> scripts/config.local.sh

Watch the nginx access log (Ctrl+C to stop):

./scripts/watch-access.sh
Streaming nginx access log from 192.168.1.50 (Ctrl+C to stop)...
─────────────────────────────────────────────────────────────────
nginx  | 10.0.0.5 - - [29/May/2026:14:02:11 +0000] "GET /FOLDER01/firmware.exe HTTP/1.1" 200 45234182
nginx  | 10.0.0.6 - - [29/May/2026:14:02:19 +0000] "GET /Catalog/Catalog.xml.gz HTTP/1.1" 200 981432

Watch the nginx error log:

./scripts/watch-errors.sh
Streaming nginx error log from 192.168.1.50 (Ctrl+C to stop)...
─────────────────────────────────────────────────────────────────
(quiet when there are no errors)

Run a firmware mirror sync (streams live threaded output):

9

Teardown

To destroy the VM and all resources Terraform created:

terraform destroy
⚠️

terraform destroy permanently deletes both VM disks, including any firmware files on the mirror data disk. Back up /opt/dell-firmware-mirror/mirror first if you want to keep the downloaded files.

To destroy and immediately re-create (useful after config changes):

terraform destroy -auto-approve && terraform apply -auto-approve
